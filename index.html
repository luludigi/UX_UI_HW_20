Project Overview

BrightSide Produce was designed to connect subscribers to their community by using BrightSide as a hub to bridge donor members and underserved, food-insecure communities. BrightSide turns fresh produce delivery subscriptions into funds that support partnerships with local stores in those communities. A nonprofit as groundbreaking as this one deserves a lot of support, and in order to increase the number of subscribers and donors, the mission needs to be crystal clear. We redesigned the BrightSide Produce nonprofit website with the goal to bring more subscribers to the grocery delivery service.

Final Thoughts

If we had more time we would do more user flow testing, especially regarding ordering deliveries and the A/B testing on CTAs and some of the navigation verbiage. We will also test task flows on actual BSP stakeholders so that we can start to zero in on solutions for any of their pain points. We also need to do a deep dive into the onboarding and checkout process. We will get a developer involved now to be sure we are moving in a direction we can build and to get any preliminary thoughts from their perspective. We also need to learn more about the actual payment process and find out how much of that is editable. Thankfully, BrightSide has been actively involved and we will be meeting with them in the coming weeks to keep this project moving.
